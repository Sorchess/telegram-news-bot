# Telegram News Aggregator Bot

Telegram-бот для агрегации новостных заголовков из нескольких первичных источников.
Поддерживает получение новостей по запросу и по подписке с автоматической доставкой.

## Возможности

- **4 новостных источника**: Reuters, BBC News, Коммерсантъ, РБК
- **По запросу**: получение заголовков из конкретного или всех источников
- **Подписки**: автоматическая рассылка новых заголовков из выбранных источников
- **Настройки**: выбор источника по умолчанию, количество заголовков
- **Логирование**: все действия пользователей сохраняются в БД

## Команды бота

| Команда | Описание |
|---------|----------|
| `/start` | Запуск бота, приветствие |
| `/help` | Справка по командам |
| `/news` | Заголовки из источника по умолчанию |
| `/news_from` | Выбрать источник и получить заголовки |
| `/news_all` | Заголовки из всех источников |
| `/sources` | Список доступных источников |
| `/settings` | Настройки (источник, кол-во заголовков) |
| `/subscribe` | Подписаться на источник |
| `/unsubscribe` | Отписаться от источника |
| `/my_subs` | Мои активные подписки |

### Таблицы

| Таблица | Назначение |
|---------|-----------|
| `sources` | Новостные источники (slug, название, URL) |
| `headlines` | Заголовки новостей с ссылками и датой публикации |
| `users` | Пользователи Telegram, их настройки |
| `subscriptions` | Подписки пользователей на источники |
| `delivery_log` | Журнал доставленных заголовков (избегает повторов) |
| `user_actions` | Аудит-лог действий пользователей |

### Индексы

- `ix_headlines_url` (UNIQUE) — дедупликация заголовков по URL
- `ix_headlines_source_published` — быстрая выборка последних заголовков по источнику
- `ix_subscriptions_user_source` (UNIQUE) — один пользователь = одна подписка на источник
- `ix_delivery_user_headline` (UNIQUE) — каждый заголовок доставляется пользователю один раз

## Технологический стек

- **Python 3.11+**, асинхронный код
- **python-telegram-bot** v21+ с Job Queue
- **PostgreSQL 16** — основная БД
- **SQLAlchemy 2.0** (asyncpg) — асинхронный ORM
- **Alembic** — миграции БД
- **aiohttp** + **feedparser** + **BeautifulSoup** — сбор новостей (RSS)
- **Pydantic Settings** — конфигурация из переменных окружения
- **Docker Compose** — развёртывание

## Запуск

### Предварительные требования

- Docker и Docker Compose

### Шаги

1. **Клонируйте репозиторий:**
   ```bash
   git clone https://github.com/Sorchess/telegram-news-bot.git
   cd telegram-news-bot
   ```

2. **Создайте файл `.env`** на основе примера:
   ```bash
   cp .env.example .env
   ```

3. **Укажите токен бота** в `.env`:
   ```
   TELEGRAM_BOT_TOKEN=your_token_from_@BotFather
   ```

4. **Запустите:**
   ```bash
   docker compose up --build
   ```

   Бот автоматически:
   - Создаст таблицы в PostgreSQL
   - Зарегистрирует все источники
   - Выполнит первичный сбор заголовков
   - Начнёт периодический сбор (каждые 5 минут)
   - Будет доставлять новые заголовки подписчикам

### Локальный запуск (без Docker)

```bash
# Установите зависимости
pip install .

# Убедитесь, что PostgreSQL запущен и доступен
# Укажите POSTGRES_HOST=localhost в .env

python -m app.main
```

## Принятые решения

1.  **RSS-фиды** вместо HTML-скраппинга — надёжнее, не ломается при изменении вёрстки,
   уважает политику сайтов.
2.  **Repository-паттерн** — отделение бизнес-логики от деталей работы с БД.
3.  **Delivery Log** — гарантирует, что подписчик не получит один и тот же заголовок дважды.
4.  **User Actions** — аудит-лог для аналитики и отладки.
5.  **Inline-клавиатуры** — удобный UX вместо текстовых команд для выбора источника.
6.  **Параметризация** — интервал скраппинга, кол-во заголовков настраиваются через `.env`
   и через команды бота.
