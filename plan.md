# Telegram бот: агрегатор новостей

Это тестовое задание на позицию python-разработчик в команду **headlines**. Мы
агрегируем важные новости и анализируем финансовые рынки для трейдеров и
инвесторов. Наш telegram канал: https://t.me/headlines_for_traders.
Вам нужно разработать telegram-бота, который будет автоматически собирать новостные заголовки
с различных источников (например, bloomberg.com, kommersant.ru) и предоставлять их
пользователям. Бот должен поддерживать два режима работы: по запросу или по подписке - как
только появляются новые заголовки.
Нам особенно важно оценить вашу способность писать надежный, легко поддерживаемый
python-код, навыки работы с базами данных и самостоятельность в принятии решений и в работе с
трудностями. Мы сознательно не выделяем каких-то ограничивающих рамок, таких как количество
и название команд бота, количество выдаваемых за раз новостных заголовков, выбор источников
данных и многое другое. Продуманность и адекватность таких параметров также будет
оцениваться.

## Задание

Итак, задание предусматривает создание telegram-бота, который:
● Позволяет по запросу получить актуальные новостные заголовки из разных источников.
● Обеспечивает подписку на автоматическую рассылку новостей из выбранных источников.
● Сохраняет заголовки, действия пользователей, подписки и остальное в базе данных.
Реализовать необходимо следующие команды бота:
● Для запуска бота, получения описания функционала и списка команд.
● Для получения новостных заголовков: из источника по умолчанию, выбранного источника
или из всех доступных.
● Для настройки получения новостных заголовков, как то: выбор источника по умолчанию,
выбор количества выдаваемых новостных заголовков за раз и всё, что вы сочтете нужным.
● Для управления подписками: просмотр активных подписок, добавление новых и их отмена.

## Детали

```
● Бот должен поддерживать минимум 3–4 новостных источника, включая как российские, так и
зарубежные.
● Получение новостных заголовков может быть реализовано через скраппинг или с помощью
публичного API.
● Источники должны быть первичными новостными ресурсами, а не агрегаторами (например,
"Новости в Дзене" не подходят).
● При отправке заголовков нужно также добавлять краткую ссылку на оригинальную новость
(например, ссылка https://kommersant.ru/doc/7553098, но отображаться должна как
kommersant.ru).
● При запросе пользователя новостные заголовки должны извлекаться из базы данных, а не
загружаться заново с оригинального источника.
```
